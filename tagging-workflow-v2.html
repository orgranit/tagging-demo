üìã Tagging System Rules

üîí Protection-Based Workflows

üîí Protected Tags
‚Ä¢ Require approval for attach/detach
‚Ä¢ Create pending requests
‚Ä¢ Modal-based approval workflow
‚Ä¢ Examples: compliance, kyc-required
‚ö° Non-Protected Tags
‚Ä¢ Apply immediately
‚Ä¢ No approval needed
‚Ä¢ Direct final state
‚Ä¢ Examples: general-use

üö´ Sanitization Rules

‚úÖ
Vault Removed from Processing: Only when capacity would be exceeded (>3 attachments per vault)
‚ùå
Individual Operations Rejected:
  • Cannot add: attachment already exists as final
  • Cannot add: pending request already exists for same vault-tag pair
  • Cannot remove: attachment does not exist
  • Cannot remove: pending request already exists for same vault-tag pair
  • Cannot process: tag does not exist or is being edited

‚öôÔ∏è Input Validation Rules

Array Constraints
‚Ä¢ Maximum 3 tags per add array
‚Ä¢ Maximum 3 tags per remove array
‚Ä¢ No duplicate tags within arrays
‚Ä¢ No tag can appear in both add and remove arrays
Tag Existence
‚Ä¢ All referenced tags must exist in tags table
‚Ä¢ Tags with 'pending edit' status are invalid
‚Ä¢ Tags being deleted are invalid
‚Ä¢ Case-sensitive tag ID matching

üìé Capacity Rules

Vault Limits
‚Ä¢ Maximum 3 final attachments per vault
‚Ä¢ Pending attachments count toward limit
‚Ä¢ Exceeding capacity removes entire vault from processing
‚Ä¢ Vault removal affects all tag operations for that vault
Examples
Capacity Exceeded:
Vault v1 has 2 final + 1 pending ‚Üí trying to add 2 more ‚Üí entire vault removed
Already Exists:
Add t1 to v1, but v1-t1 already final ‚Üí operation rejected
Protected Tag:
Add compliance tag ‚Üí creates pending request, Add general-use tag ‚Üí applied immediately

üé≠ Status Indicators

final Confirmed attachment
pending add Awaiting approval to attach
pending remove Awaiting approval to detach
pending edit Tag being modified
